# R6战术板 

纯前端、无需后端的实时协作战术板，支持自定义房间号、多人同步编辑、圆形标记、画线、文字标注、地图背景切换。

在线地址：(填写你的 GitHub Pages 链接)

## 功能

- 多人实时同步：房主创建房间，成员加入，所有操作实时同步
- 自定义房间号：输入任意房间名称即可，无需复制长串ID
- 点击放置圆形标记：可自定义名称，名称居中显示
- 战术标注工具：画线（带箭头）、文字标注、删除、撤销
- 地图背景：支持本地上传图片或输入图片URL
- 保存与加载：导出战术板为JSON文件，可随时恢复
- 导出图片：保存当前视图为PNG

## 使用方法

### 房主创建房间
1. 在左侧“自定义房间”输入框中输入房间名称（例如 r6_team）
2. 点击“创建房间 (房主)”按钮
3. 等待其他成员加入

### 成员加入房间
1. 输入与房主相同的房间名称
2. 点击“加入房间”按钮
3. 连接成功后，所有人将看到相同的画布内容

### 工具说明
- 添加圆形标记：点击“添加圆形”工具 → 点击画布 → 输入名称
- 画线：点击“画线”工具 → 点击起点 → 拖动到终点释放
- 文字标注：点击“文字”工具 → 点击画布 → 输入文字
- 删除：点击“删除”工具 → 点击对象（或选中后按Delete键）
- 撤销：点击“撤销”按钮或按Ctrl+Z
- 清空画布：点击“清空画布”（保留网格背景）

### 地图加载
- 本地上传：点击“本地上传图片”，选择地图截图
- URL加载：输入图片链接，点击“加载地图”

### 保存与导出
- 保存战术：点击“保存战术 (JSON)”，下载文件；下次点击“加载战术文件”恢复
- 导出图片：点击“导出为图片”，保存为PNG

## 技术说明

- 前端：HTML/CSS/JS + Fabric.js
- 实时通信：PeerJS (WebRTC) + 公共信令服务器，无需自建后端
- 部署：纯静态页，可托管在 GitHub Pages、Vercel、Netlify 等

所有用户需要能够访问互联网（公共信令服务器），局域网内也可正常使用。
